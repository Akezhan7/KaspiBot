### 2. **Kaspi API - метод запроса** ❌ → ✅
**Проблема:** Использовался GET запрос, Kaspi требует POST  
**Решение:** 
- Обновлен `parser/kaspi_parser.py` - добавлен метод POST
- API требует JSON body с параметрами:
  ```json
  {
    "cityId": "750000000",
    "limit": 50,
    "page": 0
  }
  ```

---

### 3. **Прокси подключение** ❌ → ✅
**Проблема:** Неправильная передача прокси в httpx (использовался `proxies=dict`)  
**Решение:**
- В httpx параметр называется `proxy` (единственное число)
- Передается как строка: `proxy="http://user:pass@host:port"`
- Обновлены:
  - `parser/kaspi_parser.py` - использует `proxy` вместо `proxies`
  - `parser/proxy_manager.py` - возвращает строку вместо словаря
