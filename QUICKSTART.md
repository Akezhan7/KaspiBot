# БЫСТРЫЙ СТАРТ

## 1. Установка зависимостей

```bash
# Создать виртуальное окружение
python -m venv venv

# Активировать
venv\Scripts\activate  # Windows

# Установить пакеты
pip install -r requirements.txt
```

## 2. Настройка .env

Скопировать `.env.example` в `.env` и заполнить:

```env
# Telegram
TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz  # От @BotFather
ADMIN_USER_IDS=123456789  # Ваш Telegram ID (получить через @userinfobot)

# Proxy
PROXY_URL=http://user:pass@host:port  # MobileProxy.Space прокси
PROXY_CHANGE_API=https://mobileproxy.space/api/change_ip  # API смены IP

# Настройки (можно оставить по умолчанию)
SCAN_INTERVAL_HOURS=6
BATCH_SIZE=50
```

## 3. Запуск

```bash
python main.py
```

## 4. Использование

### В Telegram боте:

1. **Добавить товар:**
   ```
   /add https://kaspi.kz/shop/p/товар-107664472/
   ```

2. **Посмотреть список:**
   ```
   /list
   ```

3. **Статистика:**
   ```
   /stats
   ```

4. **Принудительная проверка:**
   ```
   /scan
   ```

## Что происходит?

1. Бот автоматически проверяет все товары каждые 6 часов
2. Для каждого товара получает список продавцов с ценами
3. Если продавец новый - парсит его телефон со страницы магазина
4. Отправляет уведомление в Telegram с контактами
5. Логирует все в `logs/bot.log` и БД `data/kaspi_monitor.db`

## Важно!

- **MobileProxy обязателен** - без него Kaspi заблокирует запросы
- **Первая проверка** начнется через 6 часов после запуска (или сразу через `/scan`)
- **Телефоны кэшируются** - запрашиваются только 1 раз для нового продавца

## Устранение проблем

### Ошибка "TELEGRAM_BOT_TOKEN не установлен"
→ Проверьте `.env` файл, он должен быть рядом с `main.py`

### Ошибка подключения к прокси
→ Проверьте формат `PROXY_URL` и доступность прокси

### Бот не отвечает
→ Проверьте что ваш ID в `ADMIN_USER_IDS`

## Логи

Все логи в `logs/bot.log`:
- INFO - нормальная работа
- WARNING - небольшие проблемы (не найден телефон и т.д.)
- ERROR - серьезные ошибки

## База данных

SQLite база в `data/kaspi_monitor.db`

Можно просмотреть через:
- DB Browser for SQLite
- sqlite3 CLI: `sqlite3 data/kaspi_monitor.db`

## Масштабирование

Для добавления большего количества товаров:
1. Увеличить `BATCH_SIZE` (с осторожностью)
2. Добавить больше прокси через ротацию
3. Уменьшить задержки (риск блокировки)
