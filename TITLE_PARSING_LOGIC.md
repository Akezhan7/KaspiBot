# СИСТЕМА ПОЛУЧЕНИЯ НАЗВАНИЙ ТОВАРОВ

## 📊 Приоритеты при парсинге (в методе get_product_offers):

1. **Корень ответа API** (data.name, data.productName, data.title)
2. **Все offers по порядку** - ищем первый непустой title
3. **Другие поля первого offer** (productName, name)
4. **masterCategory** (если есть, убираем префикс "Master - ")
5. **Fallback**: "Товар {SKU}" (если ничего не нашли)

## 🔄 Обновление при сканировании:

### Условия для обновления названия:
- `product_title` отсутствует (NULL)
- `product_title == 'Без названия'`
- `product_title.startswith('Товар ')` (fallback)

### Порядок попыток обновления:
1. Из offers (productName, title, name)
2. Из product API (если доступен - сейчас 401)
3. Сохранить только если найдено нормальное название

## ✅ Примеры работы:

### Товар с пустым title у первого продавца (SKU 143381154):
```
Offers:
  [0] title: "" (пустой)
  [1] title: "Клеевой пистолет 20 w 7 мм" ← БЕРЁМ ЭТОТ
  [2] title: "Клеевой пистолет 20w + 3 кле"
  
Результат: "Клеевой пистолет 20 w 7 мм" ✅
```

### Товар со всеми пустыми title:
```
Offers:
  [0] title: "", masterCategory: "Master - Glue guns"
  [1] title: ""
  
Результат: "Glue guns" (из masterCategory) ✅
```

### Товар вообще без данных:
```
Offers: []

Результат: "Товар 143381154" (fallback) ✅
При следующем сканировании - попытка обновить
```

## 🎯 Вывод:

**ДА, система работает для ВСЕХ товаров!**

Многоуровневая система fallback гарантирует что:
- Всегда будет какое-то название
- При появлении данных - название обновится
- Логирование всех попыток для отладки
