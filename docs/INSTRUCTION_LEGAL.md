# Инструкция: Работа с юридическими заявками

> Для юристов и сотрудников юридического отдела

---

## Обзор

Когда продавец-нарушитель на Kaspi.kz не реагирует на предупреждения,
система автоматически создаёт **юридическую заявку** с полной доказательной базой.
После контрольной закупки заявка переходит в статус **«Готов к подаче иска»**.

---

## Получение уведомлений

Вы получите уведомление в Telegram, когда:
- Создана новая юридическая заявка
- Заявка готова к подаче иска (после контрольной закупки)

```
⚖️ Новая юридическая заявка #15
Продавец: ТОО "Рога и Копыта" (merchant_123)
Товаров: 3
Статус: LEGAL_REQUEST_CREATED
```

## Команды бота

### Список всех заявок

```
/legal_requests
```

Показывает список активных юридических заявок с пагинацией:
```
⚖️ Юридические заявки (стр. 1/2):

#15 | ТОО "Рога и Копыта" | LEGAL_REQUEST_CREATED | 2026-03-15
#14 | IP Иванов | READY_FOR_LAWSUIT | 2026-03-14
#13 | ТОО "Альфа" | READY_FOR_LAWSUIT | 2026-03-12
```

Для перехода между страницами: `/legal_requests 2`

### Детали заявки

```
/legal 15
```

Показывает полную карточку заявки:
```

Продавец: ТОО "Рога и Копыта"
```

### Экспорт доказательной базы

```
/export 15
```

Скачивает **ZIP-архив** со всей доказательной базой:

```
legal_request_15.zip
├── legal_request_15.json    # Структурированные данные
├── legal_request_15.csv     # Табличный формат
├── dialog_log.txt           # Полная переписка WhatsApp
├── timeline.json            # Хронология событий
└── purchase_docs/           # Фото и документы закупки
    ├── photo_1.jpg
    └── photo_2.jpg
```

Формат по умолчанию — JSON. Для CSV:
```
/export 15 csv
```

## Содержимое доказательной базы

### JSON-файл включает:

| Поле                | Описание                              |
| ------------------- | ------------------------------------- |
| `request_id`        | Номер заявки                          |
| `seller_id`         | ID продавца на Kaspi                  |
| `seller_name`       | Название магазина                     |
| `seller_phone`      | Телефон продавца                      |
| `products`          | Список товаров (название, SKU, цена)  |
| `warnings_sent`     | Даты отправки WARN1 и WARN2          |
| `dialog_messages`   | Все сообщения переписки               |
| `purchase_data`     | Данные контрольной закупки            |
| `timeline`          | Полная хронология                     |

### Лог переписки (dialog_log.txt):

```
[2026-03-15 18:00] → WARN1: Уважаемый продавец...
[2026-03-16 10:30] ← Ответ (DIDNT_KNOW): Я не знал что нарушаю...
[2026-03-16 10:31] → Авто-ответ: Благодарим за ответ...
[2026-03-17 18:00] → WARN2: Повторное уведомление...
```

- `→` — исходящие (от бота)
- `←` — входящие (от продавца)
- В скобках — тип ответа по классификации

---

## Статусы заявок

| Статус                       | Значение                             |
| ---------------------------- | ------------------------------------ |
| `LEGAL_REQUEST_CREATED`      | Заявка создана, ожидает закупку      |
| `CONTROL_PURCHASE_REQUIRED`  | Назначена контрольная закупка        |
| `READY_FOR_LAWSUIT`          | Все данные собраны, можно подавать   |

---

## Рекомендуемый порядок работы

1. Просмотрите список заявок: `/legal_requests`
2. Откройте нужную заявку: `/legal 15`
3. Убедитесь, что статус — `READY_FOR_LAWSUIT`
4. Скачайте доказательную базу: `/export 15`
5. Проверьте полноту документов в ZIP-архиве
6. Используйте данные для подготовки искового заявления
