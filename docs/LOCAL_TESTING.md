# Полное тестирование KaspiBot (локально)

> Пошаговый план — как протестить каждый этап воронки локально,
> используя Green API Developer (бесплатный) + OpenAI ($1-2).

---

## Что понадобится

| Сервис | Тариф | Стоимость |
|--------|-------|-----------|
| **Green API** | Developer (бесплатно) | $0 — до 3 контактов |
| **OpenAI** | Pay-as-you-go | **$1-2 хватит с запасом** |
| **ngrok** | Free | $0 |
| **Telegram Bot** | — | $0 |

### Почему $1-2 на OpenAI более чем достаточно

Бот использует **gpt-4o-mini** — самую дешёвую модель:
- Входящий промпт (системный + сообщение): ~800 токенов → $0.00012
- Ответ (JSON с типом): ~50 токенов → $0.00003
- **Одна классификация ≈ $0.00015** (0.015 центов)
- 100 тестовых сообщений = **$0.015** (полтора цента)
- На $1 хватит для ~6 500 классификаций
- На $2 можно полностью протестить и ещё на пару месяцев production хватит

## Этап 0: Подготовка

### 0.1 Установка зависимостей

```bash
cd D:\Works\Freelance\KaspiBot
pip install -r requirements.txt
```

Python **3.11+** (рекомендуется 3.13).

### 0.2 Заполнение `.env`

```bash
copy .env.example .env
```

Заполнить:
```env
# === ОБЯЗАТЕЛЬНЫЕ ===
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_USER_IDS=ваш_telegram_id

# === GREEN API (Developer) ===
GREEN_API_URL=https://api.green-api.com
GREEN_API_INSTANCE_ID=ваш_instance_id
GREEN_API_TOKEN=ваш_api_token

# === OPENAI ===
OPENAI_API_KEY=sk-ваш_ключ
OPENAI_MODEL=gpt-4o-mini

# === WEBHOOK ===
WHATSAPP_WEBHOOK_HOST=0.0.0.0
WHATSAPP_WEBHOOK_PORT=8443
WHATSAPP_WEBHOOK_IP_WHITELIST=

# === ТЕСТОВЫЕ ТАЙМАУТЫ (ускоренные) ===
# В production будет 24ч, но для теста ставим 2 минуты,
# чтобы не ждать сутки на каждый переход воронки
WARN1_TIMEOUT_HOURS=0.03
WARN2_TIMEOUT_HOURS=0.03
DIALOG_TIMEOUT_HOURS=0.05
ESCALATION_INTERVAL_MINUTES=1
DIALOG_TIMEOUT_CHECK_HOURS=0.02
SCAN_INTERVAL_HOURS=1
```

> **Ключевой момент**: таймауты для тестирования выставлены в минуты, а не часы.
> `0.03` часа = ~2 минуты. Так воронка пройдёт полный цикл за 10-15 минут вместо 3 дней.

### 0.3 Настройка Green API Developer

1. Зарегистрироваться на [green-api.com](https://green-api.com/)
2. Выбрать тариф **Developer** (бесплатный, до 3 контактов)
3. Создать инстанс → записать **Instance ID** и **API Token**
4. Привязать номер телефона через QR-код:
   - На телефоне: WhatsApp → Настройки → Связанные устройства → Привязать
   - Отсканировать QR на экране Green API

> **Важно**: используйте **отдельный номер** для бота, не свой личный.
> На Developer-тарифе бот сможет писать только 3 контактам — этого достаточно для теста.

### 0.4 Настройка ngrok

```bash
# Скачать: https://ngrok.com/download
# Авторизоваться (бесплатно): ngrok config add-authtoken YOUR_TOKEN
ngrok http 8443
```

Появится URL вида `https://xxxx-xx-xx.ngrok-free.app`.

В панели Green API → Настройки инстанса:
- **Notification URL**: `https://xxxx-xx-xx.ngrok-free.app/webhook`
- Отметить галочки: **Получать уведомления о входящих сообщениях**

### 0.5 OpenAI

1. Зайти на [platform.openai.com](https://platform.openai.com/)
2. Settings → Billing → Add payment method
3. Пополнить на **$2** (минимум $5 обычно, но хватит на годы)
4. API Keys → Create new → скопировать `sk-...` в `.env`

### 0.6 Запуск unit-тестов (без API)

```bash
python -m pytest tests/ -q
```

Все 200 тестов должны пройти. Они не обращаются к реальным API.

## Этап 3: Автоматическая отправка WARN1

С ускоренными таймаутами (`ESCALATION_INTERVAL_MINUTES=1`) планировщик подхватит новые воронки в течение ~1 минуты.

**Что проверить:**
- [ ] В логе: `WARN1 sent to ...`
- [ ] Админу в Telegram пришло: «⚠️ WARN1 отправлен: {магазин}»
- [ ] `/workflow <id>` показывает статус `WARN1_SENT`
- [ ] Сообщение дошло в WhatsApp (если номер продавца — один из 3 контактов Developer)

> **Если у продавца нет телефона** или нет WhatsApp — в логе будет warning,
> воронка останется в `NEW_SELLER_ATTACH`. Это нормально — нужен продавец с реальным номером.

### Ручная отправка (если нужно конкретному продавцу):
```
/warn <seller_id>
```

## Этап 4: Ответ продавца + классификация

Здесь нужно имитировать ответ продавца. Два способа:

### Способ A: Реальный ответ с телефона
Если WARN1 дошёл в WhatsApp одному из 3 тестовых контактов — отправьте ответ с этого телефона:
- `"Я не знал, уже снимаю"` → ожидается `ALREADY_REMOVED`
- `"Докажите что это ваш товар"` → ожидается `PROVE_IT`
- `"Не буду снимать"` → ожидается `WONT_REMOVE`
- `"Дайте пару дней, сниму"` → ожидается `NEED_TIME`

### Способ B: curl на webhook (без телефона)
Нужно подставить номер телефона продавца из вашей БД:

```bash
curl -X POST http://localhost:8443/webhook ^
  -H "Content-Type: application/json" ^
  -d "{\"typeWebhook\":\"incomingMessageReceived\",\"senderData\":{\"chatId\":\"77XXXXXXXXX@c.us\",\"senderName\":\"Тест\"},\"messageData\":{\"typeMessage\":\"textMessage\",\"textMessageData\":{\"textMessage\":\"Я не знал что нельзя, уже снимаю\"}}}"
```

Подставить вместо `77XXXXXXXXX` реальный номер продавца из БД.

**Что проверить:**
- [ ] В логе: `classify` → результат (тип + confidence)
- [ ] Бот отправил авто-ответ в WhatsApp
- [ ] Админу в Telegram пришло: «💬 Ответ от {магазин}: ...»
- [ ] В логе: `message_log` записано входящее + исходящее

## Этап 5: Микро-скан при «Уже снял»

Если ответ классифицирован как `ALREADY_REMOVED`:
1. Бот автоматически делает мини-скан товара через Kaspi API
2. Проверяет, действительно ли продавец отсоединился

**Что проверить:**
- [ ] В логе: `check_detachment` → результат проверки
- [ ] Если продавец реально снял — статус → `DETACHED` → `CLOSED`
- [ ] Если не снял — бот ответит «Мы проверили — товар всё ещё на карточке»

## Этап 6: Автоэскалация WARN1 → WARN2

Если продавец НЕ ответил или ответил но не снял — через ~2 мин (тестовый таймаут `WARN1_TIMEOUT_HOURS=0.03`):

**Что проверить:**
- [ ] Планировщик автоматически отправил WARN2
- [ ] В логе: `WARN2 sent to ...`
- [ ] `/workflow <id>` → статус `WARN2_SENT`
- [ ] Сообщение WARN2 строже, чем WARN1 (юридические формулировки)

## Этап 7: Эскалация WARN2 → юрзаявка

Ещё через ~2 мин без ответа:

**Что проверить:**
- [ ] В логе: `escalate_to_legal`
- [ ] `/workflow <id>` → статус `LEGAL_REQUEST_CREATED`
- [ ] В Telegram: «⚖️ Юридическая заявка #N создана»
- [ ] `/legal_requests` → заявка в списке
- [ ] `/legal <id>` → все данные заполнены (магазин, телефон, товары, таймлайн)

## Ограничения Developer-тарифа Green API

| Ограничение | Что это значит |
|-------------|---------------|
| Максимум 3 контакта | Бот может отправлять только 3 разным номерам. Если продавцов больше — WARN не отправится, в логе будет ошибка |
| Нет гарантии стабильности | Для тестов ОК, в production перейти на платный тариф ($15/мес) |
| Один инстанс | Одновременно только один привязанный номер |

**Совет для тестирования**: добавьте в мониторинг товар, где прилепала — это **ваш собственный тестовый номер** (или номер коллеги). Тогда:
1. Бот найдёт продавца при сканировании
2. Отправит WARN1 на ваш номер
3. Вы ответите с телефона — бот классифицирует и ответит
4. Полный живой E2E тест

## Типичные проблемы

| Проблема | Решение |
|----------|---------|
| `GREEN_API_INSTANCE_ID не установлен` | Заполните `.env` |
| `OPENAI_API_KEY не установлен` | Заполните `.env` — классификатор будет возвращать UNKNOWN |
| Webhook не получает сообщения | 1) Проверьте ngrok URL в панели Green API 2) Проверьте что галочки на входящие включены |
| `Developer: contact limit` | На Developer-тарифе макс. 3 контакта — это нормально |
| WARN не отправляется | У продавца нет телефона или WhatsApp на номере |
| Port 8443 занят | Измените `WHATSAPP_WEBHOOK_PORT` в `.env` |
| Тесты падают | `pip install -r requirements.txt` — обновите зависимости |
| Эскалация не срабатывает | Проверьте что таймауты маленькие (0.03) и `ESCALATION_INTERVAL_MINUTES=1` |
| `ModuleNotFoundError` | Убедитесь что запускаете из корня проекта |
| ngrok URL поменялся | Каждый перезапуск ngrok (free) даёт новый URL — обновите в Green API |
