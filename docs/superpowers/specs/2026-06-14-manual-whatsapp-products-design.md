# Ручная отправка товаров продавцу в WhatsApp

## Цель

Добавить в карточку продавца одну кнопку `Отправить товары в WhatsApp`.
Кнопка позволяет администратору немедленно отправить продавцу актуальный
список товаров, не отключая и не дублируя существующую автоматическую
воронку предупреждений.

## Пользовательский сценарий

1. Администратор открывает список продавцов и карточку нужного магазина.
2. Нажимает `Отправить товары в WhatsApp`.
3. Бот повторно получает из базы все активные товары продавца.
4. Если первое предупреждение ещё не отправлялось, бот отправляет WARN1
   существующим механизмом и тем самым убирает продавца из очереди на WARN1.
5. Если WARN1 уже отправлялся, бот отправляет короткое сообщение с актуальным
   списком названий и ссылок, не меняя текущий этап воронки.
6. Бот фиксирует время ручной отправки и количество активных товаров.
7. В карточке продавца показывается прогресс относительно последней ручной
   отправки: сколько товаров было, сколько осталось и сколько снято.

## Совместимость с автоматической воронкой

Автоматическая рассылка остаётся включённой.

- Продавец в статусе `NEW_SELLER_ATTACH` после ручного WARN1 переходит в
  `WARN1_SENT`; планировщик больше не выберет его для первого предупреждения.
- Для продавцов в более поздних статусах ручная кнопка не отправляет WARN1 или
  WARN2 повторно. Она отправляет отдельное информационное сообщение со списком
  актуальных товаров.
- Автоматические WARN2 и юридическая эскалация продолжают работать по
  существующим срокам.
- Ручная отправка учитывается существующими дневными и персональными
  антиспам-ограничениями.

## Данные

В `seller_workflows` добавляются поля:

- `manual_products_sent_at` — время последней успешной ручной отправки;
- `manual_products_initial_count` — число активных товаров на момент отправки.

Отдельная таблица не требуется: для одного активного продавца используется одна
воронка, а клиенту нужен прогресс от последнего ручного действия.

Текущее число товаров берётся из `product_sellers` с `is_active = 1`.
Количество снятых вычисляется как:

`max(initial_count - current_count, 0)`.

## Отправка сообщения

Для нового продавца используется существующий WARN1 со всеми актуальными
товарами и документами.

Для уже обработанного продавца используется отдельный короткий шаблон:

> По итогам разговора направляем актуальный список карточек, от которых
> необходимо отсоединиться:
>
> Название товара
> Ссылка

Сообщение отправляется через существующий WhatsApp-клиент и записывается в
`message_log` с отдельным кодом шаблона.

## Интерфейс

В карточке продавца:

- одна кнопка `Отправить товары в WhatsApp`;
- после успешной ручной отправки блок:
  - `Отправлено: дата и время`;
  - `Было товаров: N`;
  - `Осталось: M`;
  - `Откреплено: K`;
  - состояние `без изменений`, `частично открепился` или
    `полностью открепился`.

Кнопка доступна только администраторам. Если телефона нет, активных товаров нет
или WhatsApp отклоняет отправку, данные отслеживания не обновляются.

## Ошибки и повторные нажатия

- Нажатие кнопки сразу запускает отправку без дополнительного экрана.
- Повторная ручная отправка разрешена с учётом антиспам-ограничений.
- После успешной повторной отправки исходное количество и время заменяются
  новым снимком.
- При ошибке WhatsApp статус воронки и снимок прогресса не изменяются.
- Если активной воронки нет, она создаётся и получает все текущие товары
  продавца.

## Проверки

Нужны точечные тесты:

1. Ручная отправка новому продавцу использует WARN1 и фиксирует снимок.
2. Для уже отправленного WARN1 уходит короткий список без смены этапа.
3. Ошибка отправки не обновляет снимок.
4. Прогресс правильно считается для полного, частичного и нулевого снятия.
5. Продавец без телефона или активных товаров не получает сообщение.
